# 优化方案：目录（ToC）与特殊版式排版

## 目标

减轻翻译后目录「点线成块、章节号/标题/页码断行」等问题，提升 Markdown 可读性，且与现有分块翻译架构兼容。

## 问题复盘

- PDF 目录常为多个 **独立 text block**；`get_text("text")` 得到的行序不一定等于视觉上的「一条目录一行」。
- 现有 `collapse_toc_dot_leaders` 仅删除 **整行几乎全是点** 的长行；无法处理：
  - 「`3.1 标题` + 同行大量点」；
  - 编号、标题、页码被拆成多行；
  - 模型在译文中 **再生** 大量点线。

## 阶段 A：送入模型前的文本规范化（优先）

1. **行内引导线折叠**  
   - 用正则识别「标题 + 连续点/中点/省略号 + 可选页码」在同一物理行的情况，折叠为例如：`3.1 问答` 或 `3.1 问答 … 12`（统一约定一种格式，减少模型照抄长点串）。

2. **目录页启发式**  
   - 若一页满足：多短行、行尾多为数字（页码）、或关键词 `Contents` / `目录` 等，对该页应用 **专用规则**（甚至单独 system 补充：「输出为 Markdown 列表，不要输出引导点」）。  
   - 注意与正文页区分，避免误伤。

3. **与现有函数的关系**  
   - 在 `collapse_toc_dot_leaders` 之后或与之组合为单一 `sanitize_page_text_for_translation()`，在 `pipeline.py` 取页文本后统一调用，避免多处重复。

**可行性**：高；不依赖新库。

## 阶段 B：基于布局的同行合并（效果通常优于纯文本）

- 使用 `page.get_text("dict")` 或带 bbox 的 blocks，按 **垂直位置（y）** 分桶，将 **同一基线** 上的片段按 x 排序后拼成一行。
- 对合并后的行再套用阶段 A 的点线折叠。
- **风险**：复杂图表页可能误判；可与阶段 A 的「目录页检测」联动，仅在少数页面启用 bbox 合并以控制风险。

**可行性**：中；需针对样例 PDF 调阈值与测试。

## 阶段 C：译后清洗

- 对模型输出再次应用「删除纯点行」或与输入对称的规则，去掉译文里大段 `............`。
- 注意勿误删合法省略号段落；可与最小行长度、上下文启发式结合。

## 与 OCR 的关系

若未来启用 OCR，目录页往往同样混乱；**同一套 ToC 规范化** 应在「最终进入分块的字符串」上执行，无论文本来自 `get_text` 还是 OCR。

## 验收建议

- 使用曾出现问题的论文（如 arXiv 2310.19736v3 类目录）对比 **规范化前后** 的：行数、目录区可读性、是否误伤正文。
