## 当前 MVP 状态 - 文档整理基线（2026-06-29）

### 已完成

- Python 核心包与 CLI：`init`、`split`、`translate`、`links`、`run`。
- PDF 正文/参考文献拆分：支持 `References`、`Bibliography`、`参考文献` 等标题识别，并可用尾部比例兜底。
- 页级分块：支持 1-3 页分块和页级重叠。
- 翻译后端：`echo`、OpenAI 兼容、DeepSeek、Ollama、DeepL、hybrid。
- 记忆目录：`glossary.json`、`entities.json`、`chunk_summaries.json`、`style_notes.yaml`、`pending_review.json`、`running_summary.md`。
- 串联模式：支持前文摘要、上一块译文尾部、段尾顺延协议。
- 并联模式：支持分批并行翻译并按顺序合并。
- 精品翻译雏形：通过硅基流动巡视生成术语草稿和风险标签。
- Web 工作台：多用户登录、管理员配置、任务状态、取消任务、收藏任务、审计日志、Markdown/zip 下载。
- 前端构建：Vue 3 + Vite 构建产物由 FastAPI 静态托管。
- 文档整理：新增 `docs/` 正式文档体系，`plans/` 仅作为本地草稿来源。
- 方向重整：明确算法级翻译流程为专利主线，并补充表格/注释/人名、图像/OCR、结构分段和 HTML/PDF 输出方向。
- 文档命名：将 `docs/` 内主要文档改为中文文件名。

## 结构 IR 基础原型（2026-06-29）

### 已完成

- 新增 `pdf_translate/extractors/`，实现轻量 `DocumentIR`、`PageIR`、`BlockIR` 和基础块类型识别。
- 表格块已记录启发式行列、表头、数字 token、confidence 和 warning。
- 新增 `pdf_translate/chunkers/`，实现实验性 `StructureChunk`、Markdown table 渲染和结构分段 manifest。
- 新增 `pdf_translate/qa/`，实现 `structure_qa.json`，记录表格维度、页级风险和结构块统计。
- `pipeline.run_translate()` 会生成 `output/document_ir.json`、`output/structure_chunks_manifest.json` 和 `output/structure_qa.json`。
- CLI `translate` / `run` 增加 `--chunk-strategy page|structure`，默认仍为 `page`。
- 新增 `tests/test_structure_ir.py`，覆盖块分类、表格行列元数据、结构 chunk provenance、结构 QA 和 echo 后端最小管线。

### 验证

- `python -m unittest discover -s tests`
- `python -m compileall pdf_translate tests`

### 仍需后续

- 表格行列结构恢复仍是启发式基础，复杂跨页表、合并单元格、脚注/图注/实体归位仍需继续实现。
- 真实 OCR/VLM 调用、双语 HTML、PDF 译文输出和结构不变量 QA 尚未实现。

## 图像/OCR 路由基础（2026-06-29）

### 已完成

- `PageIR.meta` 新增文本字符数、文本区域比例、图像区域比例和块类型计数，作为页面风险判断的本地事实。
- 新增 `pdf_translate/vision/`，基于 DocumentIR 生成页面级 `vision_route.json`。
- `vision_route.json` 会输出 `text_only`、`local_ocr`、`vlm_review`、`skip_blank` 建议、风险等级、风险分数、触发原因和下一步说明。
- `pipeline.run_translate()` 每次翻译都会写出 `output/vision_route.json`，为后续 OCR/VLM 接入、实验指标和专利证据链提供记录。
- 新增单元测试覆盖低文本图片页路由到 `local_ocr`，并验证 echo 后端最小管线写出 `vision_route.json`。

### 验证

- `python -m unittest discover -s tests`
- `python -m compileall pdf_translate tests`

### 仍需后续

- 尚未接入真实本地 OCR 引擎、页面截图、图片区域裁剪或 VLM 调用。
- 路由阈值仍是保守启发式，需要在真实论文样本上统计误报、漏报和图像/扫描页覆盖率。
