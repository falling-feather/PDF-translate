## 当前 MVP 状态 - 文档整理基线（2026-06-29）

### 已完成

- Python 核心包与 CLI：`init`、`split`、`translate`、`links`、`run`。
- PDF 正文/参考文献拆分：支持 `References`、`Bibliography`、`参考文献` 等标题识别，并可用尾部比例兜底。
- 页级分块：支持 1-3 页分块和页级重叠。
- 翻译后端：`echo`、OpenAI 兼容、DeepSeek、Ollama、DeepL、hybrid。
- 记忆目录：`glossary.json`、`entities.json`、`chunk_summaries.json`、`style_notes.yaml`、`pending_review.json`、`running_summary.md`。
- 串联模式：支持前文摘要、上一块译文尾部、段尾顺延协议。
- 并联模式：支持分批并行翻译并按顺序合并。
- 精品翻译雏形：通过硅基流动巡视生成术语草稿和风险标签。
- Web 工作台：多用户登录、管理员配置、任务状态、取消任务、收藏任务、审计日志、Markdown/zip 下载。
- 前端构建：Vue 3 + Vite 构建产物由 FastAPI 静态托管。
- 文档整理：新增 `docs/` 正式文档体系，`plans/` 仅作为本地草稿来源。
- 方向重整：明确算法级翻译流程为专利主线，并补充表格/注释/人名、图像/OCR、结构分段和 HTML/PDF 输出方向。
- 文档命名：将 `docs/` 内主要文档改为中文文件名。

## 图注表注脚注归属基础（2026-06-29）

### 已完成

- `pdf_translate.extractors.document_ir.assign_block_parents()` 新增同页关系归属规则。
- 英文 `Fig. 1` / `Figure 1`、`Table 1` 和中文 `图1` / `表1` 类 caption 会被识别为图注或表注。

- `structure_qa.json` 新增 `relationships` 明细，并在 summary 中统计 `caption_count`、`caption_linked_count`、`caption_orphan_count`、`footnote_count`、`footnote_linked_count`、`footnote_orphan_count`、`table_footnote_count`、`relationship_count` 和 `relationship_warning_count`。
