## 手工回归场景

| 场景 | 输入 | 期望 |
| --- | --- | --- |
| 最小流程 | 1 个文本型 PDF，后端 `echo` | 任务完成，生成 Markdown、links CSV、zip |
| 串联翻译 | 3 页以上论文，重叠 1 页 | 有前文摘要与尾部衔接，无明显重复 |
| 并联翻译 | 多块文本，workers > 1 | 输出顺序正确，段尾顺延文件为空 |
| 精品翻译 | 已配置硅基巡视模型 | `output/survey/*.json` 生成，术语写入 glossary |
| 取消任务 | 长任务运行中点击终止 | 当前块结束后状态为 cancelled，可下载已译部分 |
| 管理端 | 管理员配置 DeepSeek 与后端开关 | 用户端可选后端与管理端一致 |
| 权限 | 普通用户访问他人任务 | 返回 404 或无权访问 |
| 表格密集论文 | 含多列表格、脚注、单位和显著性符号 | 表格行列可读，数字和单位不丢失 |
| 图像/扫描页论文 | 含低文本页、图片型表格或扫描页 | 生成 OCR/VLM 风险记录、页面预览、候选区域裁剪证据和 OCR 调度任务，低文本页不被静默跳过 |
| 跨页内容 | 跨页段落、续表、续图注 | 分段结果不产生明显半句切断和重复译文 |
| 双语 HTML | 任意文本型论文 | 原文、译文、QA 问题和修复建议能按块同屏对照 |
| PDF 输出原型 | 复杂表格和图注论文 | PDF 译文能保留标题、段落、表格、图注和页码关系 |

## 翻译质量指标

用于规划、专利原型和导师评审：

- 机器可读实验摘要：每次任务保存 `output/experiment_metrics.json`，并回链 `structure_qa.json`、`structure_chunks_manifest.json`、`table_reconstruction.json`、`structure_hints_manifest.json`、`chunk_boundary_qa.json`、`chunk_strategy_comparison.json`、`vision_route.json`、`vision_pages/`、`vision_crops/`、`ocr_tasks.json`、`ocr_results.json`、`ocr_writeback.json`、`ocr_candidate_qa.json`、`ocr_candidate_promotion.json`、`document_ir_ocr.json`、`document_ir_promoted.json`、`qa_report.json`、`repair_plan.json`、`repair_requests.json`、`repair_results.json`、`repair_validation.json`、`repair_merge.json`、`repair_merge_qa.json`、`run_metrics.json`、`run_log.jsonl` 和 `cost_estimate.json`。
- 批量实验摘要：`pdf-translate experiment` 会生成 `batch_experiment_manifest.json`、`batch_experiment_summary.json`、`batch_experiment_summary.md` 和 `batch_experiment_review.csv`；可通过 `--sample-manifest` 读取 `source_pdf`、`sample_id`、`pdf_type`、`tags`、`notes`，其中 `source_pdf` 支持相对样本清单的路径，缺省时按 PDF 入参顺序补位，并为每个样本/策略保留续表拒绝类别、`human_score`、`reviewer`、`review_notes`。
- 运行性能摘要：每次任务保存 `output/run_metrics.json`，记录总耗时、阶段耗时、翻译请求次数、HTTP 尝试次数、HTTP 重试次数、请求字符数、译文字符数、估算 token、跳过块和吞吐指标；`output/run_log.jsonl` 保留事件级明细。
- 成本估算摘要：每次任务保存 `output/cost_estimate.json`，记录成本画像是否命中、币种、单位价格、估算 token/字符/请求用量、可计费请求数和估算金额；未配置付费后端价格时，不把金额 0 解释为真实免费。
- 漏译率：源段落或结构块未在译文中对应出现的比例。
- 重复译文率：由重叠页或合并造成的重复片段比例。
- 术语一致率：术语库中的术语在译文中使用一致的比例。
- 数字保留率：源文数字集合在译文中保留的比例。
- 公式/变量保留率：公式编号、变量名、LaTeX 片段保留比例。
- 引用保留率：`[1]`、`(Smith, 2024)`、图表编号等引用保留比例。
- 表格行列保持率：输出表格维度与源表格维度一致的比例。
- 表格可重建率：`reconstructable_table_count / table_count`，用于观察源表格能否被本地结构层恢复为可消费单元格。
- 合并单元格候选率：`table_merged_cell_candidate_count / table_count`，用于观察缺列行、标题行或内部空单元格等疑似合并单元格现象在样本中的密度。
- Ragged 表格率：`table_ragged_table_rate`，用于观察表格抽取后行长不一致的比例，辅助区分真实合并单元格和抽取噪声。
- 表格空单元格率：`table_empty_cell_rate`，用于观察表格结构中空位密度，辅助定位 rowspan/colspan 候选和 OCR/解析缺口。
- 合并单元格候选类型分布：`experiment_metrics.breakdowns.table_merged_cell_candidate_type_counts`，用于区分 `rowspan`、`colspan` 等候选结构。
- 结构提示覆盖率：`structure_hint_chunk_count / chunk_count`，用于观察表格、续表、脚注绑定、合并候选等结构证据实际进入翻译请求的 chunk 比例。
- 结构提示长度：`structure_hint_char_count`、`structure_hint_avg_char_count`、`structure_hint_max_char_count`，用于观察结构保护提示对上下文预算的占用。
- 结构提示内合并候选分布：`experiment_metrics.breakdowns.structure_hint_merged_cell_candidate_type_counts` 与 `structure_hint_merged_cell_candidate_reason_counts`，用于观察真正注入翻译控制的合并候选类型和原因，而不只统计源表格候选总量。
- 表格数字单元格率：含数字 token 的表格单元格比例，用于衡量表格保护翻译中需要锁定的关键单元格密度。
- 表格脚注单元格绑定率：`table_footnote_cell_binding_count / table_footnote_binding_count`，用于观察已归属表格脚注中有多少能进一步回到具体单元格。
- 表格脚注未绑定率：`table_footnote_unbound_count / table_footnote_binding_count`，用于观察脚注标记缺失、普通数字/字母歧义或复杂排版导致的单元格级归属失败比例。
- 表格形状异常率：`table_shape_error_count / source_table_count`，用于观察表格被线性化或行列丢失的比例。
- 表格单元格 token 异常率：`table_cell_token_error_count / table_numeric_cell_count`，用于观察关键数值单元格被错误改写或拆散的比例。
- 表格锁定 token 缺失率：`missing_table_locked_token_count / source_table_locked_token_count`，用于观察数字、单位和显著性符号是否保留在对应单元格。
- 修复请求就绪率：`repair_backend_request_count / repair_request_count`，用于观察 QA 问题中有多少可直接交给翻译后端进入局部重译。
- 修复执行成功率：`repair_succeeded_count / repair_executed_request_count`，用于观察已执行修复请求是否稳定产出候选修复片段。
- 修复验证通过率：`repair_validation_passed_count / (repair_validation_passed_count + repair_validation_failed_count)`，用于观察候选修复片段是否满足本地不变量门禁。
- 修复锁定 token 通过率：`1 - repair_validation_missing_locked_token_count / repair_validation_checked_locked_token_count`，用于观察局部修复是否真的补回数字、单位、实体或其他锁定 token。
- 修复合并应用率：`repair_merge_applied_count / repair_merge_candidate_count`，用于观察验证通过的候选修复有多少能被安全写入独立合并副本。
- 合并后 issue 降幅：`post_repair_issue_delta / translation_issue_count`，用于观察修复副本相对原译文 QA 是否减少问题。
- 续表识别率：跨页续表边界能被 `table_continuations` 识别并被结构分段保护的比例。
- 续表链合并率：`table_chain_merged_count / table_chain_candidate_count`，用于观察跨页表格链有多少能通过兼容性闸门形成合并重建视图。
- 续表链拒绝率：`table_chain_reject_count / table_chain_candidate_count`，用于观察疑似续表中有多少因列数漂移、表头突变或低置信风险被阻止强合并。
- 续表拒绝原因分布：`experiment_metrics.breakdowns.table_chain_reject_reason_category_counts`，用于观察续表候选被拒绝的主要类别，例如表头突变、列数漂移或不可重建片段。
- 续表解释密度：`table_chain_reject_reason_per_rejected_chain` 与 `table_chain_warning_reason_per_candidate_chain`，用于观察续表合并闸门是否为拒绝或低置信合并留下足够解释证据。
- 续表链行增益：`table_chain_row_gain`，用于观察合并续表后比首段表格新增的有效行数。
- 图注/脚注归位率：图注、表注、脚注能回到所属图表、表格、段落或页面的比例。
- 跨页父子关系成功率：`experiment_metrics.rates.cross_page_parent_success_rate`，用于观察同页归属失败后的一页跨页回退是否能找到可信父块。
- caption/footnote 跨页链接率：`caption_cross_page_link_rate` / `footnote_cross_page_link_rate`，用于区分图表说明与脚注在跨页场景中的归位效果。
- 实体候选覆盖率：作者、人名、机构、数据集、模型名被本地规则识别为候选实体的比例。
- 人名/机构保留率：候选或确认实体在译文中被保留或一致翻译的比例。
- 实体缺失数：源文候选实体未在译文中保留的数量。
- 图像/扫描页覆盖率：需要 OCR/VLM 的页面被识别和处理的比例。
- 图像预览证据覆盖率：`vision_preview_page_count / page_count`，用于观察被路由页面是否已经生成可人工复核的页面预览。
- 图像候选区域裁剪密度：`vision_region_crop_count / routed_page_count`，用于观察路由页是否已经给出细粒度 OCR/VLM 候选区域。
- OCR 任务密度：`ocr_task_count / routed_page_count`，用于观察路由页是否已经转成可执行 OCR/VLM 待办。
- OCR 任务就绪率：`ocr_ready_task_count / ocr_task_count`，用于观察调度任务是否具备页面预览或区域裁剪输入图。
- OCR 结果载荷有效率：`ocr_result_payload_count / (ocr_result_payload_count + ocr_invalid_result_count)`，用于观察外部或本地 OCR 执行器返回的结果文件是否被规范化为有效结果。
- OCR 本地执行成功率：`ocr_executor_succeeded_task_count / ocr_executor_attempted_task_count`，用于观察可选本地 OCR 执行器在已有输入图任务上的成功情况。
- OCR 结果覆盖率：`ocr_result_count / ocr_task_count`，用于观察待执行任务中已有多少获得 OCR 结果载荷。
- OCR 结果接受率：`ocr_accepted_result_count / ocr_result_count`，用于观察 OCR 结果通过状态、文本和置信度门禁的比例。
- OCR 回写应用率：`(ocr_block_writeback_count + ocr_page_writeback_count) / ocr_task_count`，用于观察 OCR 候选文本是否已经进入增强 DocumentIR。
- OCR 候选可晋级率：`ocr_candidate_promotable_count / ocr_candidate_qa_count`，用于观察回写候选中有多少通过本地 QA 门禁。
- OCR 候选实际晋级率：`ocr_candidate_promoted_count / ocr_candidate_qa_count`，用于观察有多少 OCR 文本进入 `document_ir_promoted.json` 并参与结构分段。
- OCR 候选合格晋级率：`ocr_candidate_promoted_count / ocr_candidate_promotion_eligible_count`，用于观察通过 QA 的候选是否能定位到原始回写目标并被写入晋级 IR。
- 页面交接碎片率：分页造成半句、跨页段落、续表被切断的比例。
- 边界切开率：已识别页面边界碎片中仍被当前 chunks 切开的比例。
- 边界保护率：已识别页面边界碎片中被结构分段器显式保护的比例。
- 边界切开降幅：结构分段相对固定页分段减少的切开边界数量和比例。
- 预算切分原因分布：`chunk_boundary_qa.summary.budget_split_reason_counts`，用于观察 chunk 是因目标预算、硬预算、页数阈值、结构关系还是文档结束而切分。
- 预算压力分布：`chunk_boundary_qa.summary.budget_pressure_counts`，用于观察 chunk 处于目标预算内、超过目标预算或超过硬预算的比例。
- 预算越界块率：`experiment_metrics.rates.budget_overflow_chunk_rate`，用于观察结构关系保护是否频繁推高单块大小。
- 结构关系保护次数：`experiment_metrics.quality.structural_relation_protected_count`，用于观察图注、表注、脚注与父块是否被结构分段器保护在同一块内。
- 英文残留比例：译文中非术语英文残留的比例。
- 单篇运行耗时：`run_metrics.summary.total_elapsed_ms`，用于不同流程版本的端到端性能对比。
- 阶段耗时：`run_metrics.summary.stage_elapsed_ms`，用于定位 PDF 抽取、结构分析、翻译、合并、QA 和修复中最慢的阶段。
- 翻译请求次数：`run_metrics.summary.translation_request_count`，用于估算 API 调用规模。
- HTTP 尝试与重试次数：`run_metrics.summary.http_attempt_count`、`run_metrics.summary.http_retry_count`、`experiment_metrics.performance.http_retry_count`，用于观察限流、断连和服务端错误对成本/耗时的影响。
- 可计费请求数：`cost_estimate.usage.billable_request_count` 和 `experiment_metrics.performance.billable_request_count`，用于在存在 HTTP 重试时估算固定请求成本。
- 请求/译文字符数与估算 token：`request_char_count`、`translated_char_count`、`estimated_request_token_count`、`estimated_translated_token_count`，用于接入后端价格表。
- 估算成本：`cost_estimate.summary.estimated_total_cost` 和 `experiment_metrics.performance.estimated_total_cost`，用于不同流程版本的费用对比。
- 输出可读性评分：Markdown、HTML、PDF 三类输出分别记录 1-5 分。
- 人工可读性评分：1-5 分，记录评分人和样本。

## 发布前检查清单

- README 能导航到 docs、安装说明和测试指南。
- 新增功能有最小测试或明确人工验收路径。
- 外部 API 失败时不会导致任务状态卡死。
- 管理员配置与用户端可见后端一致。
- 上传文件、输出路径、zip 打包路径都限制在任务目录。
- 完成的规划项已迁入发布历史。
- 专利原型相关实验保存配置、指标、样本类型和结果。
