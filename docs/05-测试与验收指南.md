## 最低自动化测试

第一批测试不依赖真实翻译 API，优先覆盖纯逻辑：

| 编号 | 测试对象 | 验收点 |
| --- | --- | --- |
| QA-REF-01 | `pdf_structure.py` | 能识别 `References`、`Bibliography`、`参考文献` 等标题；无标题时按配置决定是否尾部兜底 |
| QA-CHUNK-01 | `chunking.py` / `chunkers/structure.py` / `qa.chunk_boundary` | `pages_per_chunk` 与 `overlap_pages` 组合正确，非法参数报错；结构分段能保护高风险页面边界，分段边界 QA 能统计保护/切开情况，并能与固定页分段做同文档对比 |
| QA-MERGE-01 | `pipeline_merge.py` | 合并时剥离 chunk YAML front matter，顺序稳定 |
| QA-DEFER-01 | `deferral_markers.py` | 段尾顺延标识解析、替换、清理行为稳定 |
| QA-MEM-01 | `memory_store.py` | 术语合并按英文去重，术语冲突进入待审，摘要追加不破坏 JSON |
| QA-FILE-01 | `export_filename.py` | 中文友好下载名、部分译文名、zip 名生成正确 |
| QA-JOB-01 | Web 任务 | 创建、取消、完成、失败状态可恢复并有审计记录 |
| QA-IR-01 | DocumentIR | 表格、续表、图注、表格脚注、脚注、公式、图片、人名机构和模型/数据集名具有类型、候选实体或父子关系，能输出页面边界碎片风险 |
| QA-TABLE-01 | 表格重建证据与提示注入 | `table_reconstruction.json` 能输出表格单元格坐标、表头/行头、数字/单位/显著性 token、表注/脚注绑定、续表关系和可重建率；当前 chunk 的表格证据能生成「结构保护提示」并进入翻译请求；译后 QA 能定位表格单元格锁定 token 缺失 |
| QA-VISION-01 | 图像/OCR 路由 | 低文本、图片密集、图注相关页面能产出本地 OCR/VLM 路由建议和原因 |
| QA-TRANS-01 | 译后规则 QA | 能检查漏数字、漏引用、图表编号、Markdown/DocumentIR 表格形状、表格单元格 token 缺失、术语一致性、术语冲突、实体候选保留、重复段落和英文残留 |
| QA-REPAIR-01 | 局部修复计划、请求、结果、验证与合并 | QA 问题能映射为带优先级、范围、动作、执行器和证据的修复项；表格单元格 token 缺失能进入 `repair_table_cell_tokens`；修复项能生成包含源文、当前译文、锁定 token 和后端 payload 的 `repair_requests.json`；修复执行阶段能生成 `repair_results.json`，默认不覆盖正式译文；候选修复片段能生成 `repair_validation.json`，记录锁定 token 与表格形状验证状态；验证通过且可安全定位的候选能进入 `repair_merge.json`、`repaired_full.md` 和 `repair_merge_qa.json` |
| QA-METRICS-01 | 实验指标汇总 | `experiment_metrics.json` 能聚合结构 QA、OCR/VLM 路由、译后 QA、修复计划、修复请求、修复结果、修复验证、修复合并和合并后 QA 的关键指标 |
| QA-OUT-01 | 输出导出 | 结构化 Markdown、双语 HTML、PDF 原型不会丢失表格、QA 标记和图注关系 |

## 手工回归场景

| 场景 | 输入 | 期望 |
| --- | --- | --- |
| 最小流程 | 1 个文本型 PDF，后端 `echo` | 任务完成，生成 Markdown、links CSV、zip |
| 串联翻译 | 3 页以上论文，重叠 1 页 | 有前文摘要与尾部衔接，无明显重复 |
| 并联翻译 | 多块文本，workers > 1 | 输出顺序正确，段尾顺延文件为空 |
| 精品翻译 | 已配置硅基巡视模型 | `output/survey/*.json` 生成，术语写入 glossary |
| 取消任务 | 长任务运行中点击终止 | 当前块结束后状态为 cancelled，可下载已译部分 |
| 管理端 | 管理员配置 DeepSeek 与后端开关 | 用户端可选后端与管理端一致 |
| 权限 | 普通用户访问他人任务 | 返回 404 或无权访问 |
| 表格密集论文 | 含多列表格、脚注、单位和显著性符号 | 表格行列可读，数字和单位不丢失 |
| 图像/扫描页论文 | 含低文本页、图片型表格或扫描页 | 生成 OCR/VLM 风险记录，低文本页不被静默跳过 |
| 跨页内容 | 跨页段落、续表、续图注 | 分段结果不产生明显半句切断和重复译文 |
| 双语 HTML | 任意文本型论文 | 原文、译文、QA 问题和修复建议能按块同屏对照 |
| PDF 输出原型 | 复杂表格和图注论文 | PDF 译文能保留标题、段落、表格、图注和页码关系 |

## 翻译质量指标

用于规划、专利原型和导师评审：

- 机器可读实验摘要：每次任务保存 `output/experiment_metrics.json`，并回链 `structure_qa.json`、`table_reconstruction.json`、`chunk_boundary_qa.json`、`chunk_strategy_comparison.json`、`vision_route.json`、`qa_report.json`、`repair_plan.json`、`repair_requests.json`、`repair_results.json`、`repair_validation.json`、`repair_merge.json` 和 `repair_merge_qa.json`。
- 漏译率：源段落或结构块未在译文中对应出现的比例。
- 重复译文率：由重叠页或合并造成的重复片段比例。
- 术语一致率：术语库中的术语在译文中使用一致的比例。
- 数字保留率：源文数字集合在译文中保留的比例。
- 公式/变量保留率：公式编号、变量名、LaTeX 片段保留比例。
- 引用保留率：`[1]`、`(Smith, 2024)`、图表编号等引用保留比例。
- 表格行列保持率：输出表格维度与源表格维度一致的比例。
- 表格可重建率：`reconstructable_table_count / table_count`，用于观察源表格能否被本地结构层恢复为可消费单元格。
- 表格数字单元格率：含数字 token 的表格单元格比例，用于衡量表格保护翻译中需要锁定的关键单元格密度。
- 表格形状异常率：`table_shape_error_count / source_table_count`，用于观察表格被线性化或行列丢失的比例。
- 表格单元格 token 异常率：`table_cell_token_error_count / table_numeric_cell_count`，用于观察关键数值单元格被错误改写或拆散的比例。
- 表格锁定 token 缺失率：`missing_table_locked_token_count / source_table_locked_token_count`，用于观察数字、单位和显著性符号是否保留在对应单元格。
- 修复请求就绪率：`repair_backend_request_count / repair_request_count`，用于观察 QA 问题中有多少可直接交给翻译后端进入局部重译。
- 修复执行成功率：`repair_succeeded_count / repair_executed_request_count`，用于观察已执行修复请求是否稳定产出候选修复片段。
- 修复验证通过率：`repair_validation_passed_count / (repair_validation_passed_count + repair_validation_failed_count)`，用于观察候选修复片段是否满足本地不变量门禁。
- 修复锁定 token 通过率：`1 - repair_validation_missing_locked_token_count / repair_validation_checked_locked_token_count`，用于观察局部修复是否真的补回数字、单位、实体或其他锁定 token。
- 修复合并应用率：`repair_merge_applied_count / repair_merge_candidate_count`，用于观察验证通过的候选修复有多少能被安全写入独立合并副本。
- 合并后 issue 降幅：`post_repair_issue_delta / translation_issue_count`，用于观察修复副本相对原译文 QA 是否减少问题。
- 续表识别率：跨页续表边界能被 `table_continuations` 识别并被结构分段保护的比例。
- 图注/脚注归位率：图注、表注、脚注能回到所属图表、表格、段落或页面的比例。
- 实体候选覆盖率：作者、人名、机构、数据集、模型名被本地规则识别为候选实体的比例。
- 人名/机构保留率：候选或确认实体在译文中被保留或一致翻译的比例。
- 实体缺失数：源文候选实体未在译文中保留的数量。
- 图像/扫描页覆盖率：需要 OCR/VLM 的页面被识别和处理的比例。
- 页面交接碎片率：分页造成半句、跨页段落、续表被切断的比例。
- 边界切开率：已识别页面边界碎片中仍被当前 chunks 切开的比例。
- 边界保护率：已识别页面边界碎片中被结构分段器显式保护的比例。
- 边界切开降幅：结构分段相对固定页分段减少的切开边界数量和比例。
- 英文残留比例：译文中非术语英文残留的比例。
- 输出可读性评分：Markdown、HTML、PDF 三类输出分别记录 1-5 分。
- 人工可读性评分：1-5 分，记录评分人和样本。

## 发布前检查清单

- README 能导航到 docs、安装说明和测试指南。
- 新增功能有最小测试或明确人工验收路径。
- 外部 API 失败时不会导致任务状态卡死。
- 管理员配置与用户端可见后端一致。
- 上传文件、输出路径、zip 打包路径都限制在任务目录。
- 完成的规划项已迁入发布历史。
- 专利原型相关实验保存配置、指标、样本类型和结果。
