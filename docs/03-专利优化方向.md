## 重要说明

本文是工程与方案梳理，不等同于专利法律意见。真正申请前，需要专利代理师围绕新颖性、创造性、实用性做检索、交底书整理和权利要求撰写。

## 基本判断

如果只把项目描述为“PDF 翻译工具”“AI 文献翻译方法”“使用术语表的翻译系统”或“保留格式的文档翻译方法”，很容易落入已有技术。更适合收束为一个具体技术方案：

> 针对学术 PDF 长文翻译中的上下文压缩、术语漂移、结构错乱和表格线性化问题，构建一种结构感知的层级式翻译控制方法：先将文档解析为带类型与边界的中间结构，再按结构原子和上下文预算生成翻译块，翻译过程中动态更新术语、摘要和衔接记忆，并通过结构不变量进行质量检测与局部重译修复。

当前更推荐把“算法级翻译流程”作为专利主线，而不是把“完全不借助 AI 的本地翻译工具”作为主线。OCR、图像识别、本地模型和 PDF 输出应作为支撑模块，服务于结构识别、低依赖运行和结果可读性。

## 当前核心问题

| 问题 | 当前表现 | 专利化切入点 |
| --- | --- | --- |
| 表格线性化 | 表格文字被抽成普通文本，行列、表头、单位和脚注丢失 | 表格结构保护、单元格上下文构造、表格维度 QA |
| 注释/人名/图注归属丢失 | 脚注、图注、人名机构和正文混在一起 | 基于结构类型和父子关系的内容归位 |
| 图像与扫描页处理不足 | 仅靠文本数量估计是否需要图像模型，容易误判 | 本地 OCR + 页面截图 + 图像区域 + VLM 路由 |
| 分页切块造成碎片 | 页末半句、跨页表格、跨页图注被拆开 | 结构单元分段和跨块顺延记忆 |
| Markdown 可读性不足 | 复杂表格、脚注、图文混排在 Markdown 中表现差 | 结构化 Markdown、双语 HTML、PDF 译文生成 |

## 现有技术拥挤区

以下方向不宜单独作为核心创新点：

- 机器翻译结合术语表或翻译记忆。
- 普通上下文感知翻译记忆。
- 文档翻译后保留样式或原位置回填。
- 单纯把表格单元格拿出来翻译。
- 单纯保留 PDF 原排版、公式和图表。

这些能力可以作为系统组成部分，但不应作为唯一创新点。

## 建议核心名称

- 一种面向学术 PDF 的结构感知层级式机器翻译方法及系统。
- 一种基于结构中间表示和动态记忆反馈的学术 PDF 分层翻译方法。
- 一种基于结构不变量检测与局部修复的学术文档翻译方法。
- 一种基于本地结构识别与结构单元分段的学术 PDF 翻译排版方法。

## 建议技术流程

```text
输入 PDF
  -> 本地结构解析与 OCR 风险检测
  -> 生成轻量 DocumentIR
  -> 译前巡视：术语、章节、表格/公式/图注/扫描页风险
  -> 用户确认术语与策略
  -> 结构单元分段：段落、表格、公式、图注、脚注不可切分
  -> 分层翻译：正文、表格、图注、公式附近文本、注释分别处理
  -> 动态记忆更新：术语、实体、摘要、尾部衔接
  -> 结构不变量 QA：数字、公式、引用、表格维度、术语一致性
  -> 局部修复：重译高风险块、段落或单元格
  -> 输出结构化 Markdown、双语 HTML、PDF 译文、QA 报告和资料包
```

## 可形成权利要求的组合方向

### 方向 A：结构感知层级翻译控制

**核心组合**：PDF 结构解析、结构原子标注、自适应分块、块间摘要、顺延记忆、翻译后记忆回流。

**技术效果**：
- 降低长上下文压缩导致的漏译。
- 降低跨块指代断裂。
- 降低术语漂移。
- 保持任务可恢复、可局部重译。

### 方向 B：表格保护式学术翻译

**核心组合**：表格结构识别、单元格上下文构造、数字/单位/符号锁定、表格维度 QA、异常单元格局部重译。

**技术效果**：
- 避免表格被线性化后语义错乱。
- 保持行列关系、表头和单位上下文。
- 提高表格短文本翻译准确性。

### 方向 B2：注释、图注与实体归位

**核心组合**：脚注/页下注释检测、图注/表注归属、人名/机构/数据集/模型名实体识别、正文与注释分层输出。

**技术效果**：
- 减少注释混入正文造成的阅读断裂。
- 保留人名、机构和专名上下文。
- 提高 Markdown、HTML 和 PDF 输出的可读性。

### 方向 C：动态记忆反馈式文档翻译

**核心组合**：译前术语候选、用户确认或置信度标记、翻译中术语回流、按页码/章节/结构类型选择性注入、冲突检测。

**技术效果**：
- 越翻译越稳定。
- 减少专业术语前后不一致。
- 降低上下文窗口浪费。

### 方向 D：结构不变量驱动的局部修复

**核心组合**：从源 DocumentIR 提取不变量、译后比对、定位异常块、选择修复粒度、只重译局部。

**技术效果**：
- 降低修复成本。
- 提高译文完整性。
- 避免全文重译引入新不稳定。

### 方向 E：图像页与 OCR/VLM 路由

**核心组合**：文本层统计、页面截图、图片区域检测、OCR 置信度、图注识别、VLM 调用决策。

**技术效果**：
- 避免扫描页和图片型表格漏译。
- 减少对图像模型的无效调用。
- 让图像处理结果回流到结构 IR，而不是只生成零散描述。

## 最建议优先落地的专利原型

优先做 “A + B + B2 + D”，并把 “E” 作为支撑能力的最小闭环：

```text
DocumentIR
  -> 表格保护
  -> 注释/图注/实体归位
  -> 图像/OCR 风险路由
  -> 结构单元分段
  -> 动态记忆注入
  -> 结构不变量 QA
  -> 局部修复
  -> 双语 HTML / PDF 译文输出
```

原因：

- 与现有项目距离最近，不需要推翻当前管线。
- 能直接回应固定页分块和表格错乱问题。
- 比单纯术语记忆更有技术形态。
- 更容易用实验数据证明效果。
- 能把本地 OCR 和 PDF 输出纳入算法流程，而不是另起一条孤立路线。

## 原型模块要求

### DocumentIR

建议最小结构：

```json
{
  "doc_id": "paper-id",
  "pages": [
    {
      "page_no": 1,
      "blocks": [
        {
          "block_id": "p1-b3",
          "type": "paragraph | heading | table | table_cell | caption | footnote | formula | reference | person | organization | header_footer | image",
          "bbox": [0, 0, 0, 0],
          "text": "...",
          "parent_id": null,
          "order": 3,
          "locked_tokens": ["(1)", "F1-score", "Table 2"]
        }
      ]
    }
  ]
}
```

验收：同一份 IR 能支撑分块、翻译、QA、双语输出和局部修复。

### 表格保护

必须保留：

- 行列结构。
- 表头、行名、单位、脚注。
- 数字、显著性符号、公式片段。
- 单元格上下文提示。
- 译后行列数和数字集合比对。

验收：表格密集论文的表格行列保持率可统计，异常能定位到表格或单元格。

### 结构感知分块

分块依据：

- 结构原子不可切分。
- 模型上下文预算。
- 章节和段落边界。
- 表格/公式/图注风险。
- 脚注、注释和人名机构上下文。
- 跨页续段和续表。
- 跨块顺延记忆。

验收：相比固定页滑窗，重复译文、半句切断、术语漂移和漏译风险下降。

### QA 与局部修复

优先做规则 QA：

- 数字集合不变量。
- 公式/变量名不变量。
- 引用编号不变量。
- 表格维度不变量。
- 图表编号不变量。
- 英文残留比例。
- 术语一致性。
- 重复译文检测。

验收：QA 能输出 `qa_report.md`，并能给出重译块、段落或单元格建议。

当前原型已具备 `structure_qa.json` 页面边界碎片检测，可记录相邻页面首尾块的断句、续表和接续风险；结构分段器已能消费该风险，在高风险相邻页面之间保护分块边界，并在 manifest 中记录 `boundary_fragment_ids`；也已具备 `qa_report.json` / `qa_report.md` 基础：数字、引用、图表编号、表格 Markdown 形状、术语一致性、术语冲突、重复段落和英文残留可在本地规则中检查；并已具备 `repair_plan.json` / `repair_plan.md`，可把 QA 问题映射为块级、段落级、表格级或术语级修复建议；同时已具备 `bilingual.html`，用于同屏审阅原文、译文、QA 问题和修复建议。公式细粒度比对、术语确认台/作用域管理、段落/表格/图注级精细边界合并和自动局部重译仍属于后续扩展。

## 申请前证据清单

建议准备测试集：

- 10 篇普通英文论文。
- 10 篇表格密集论文。
- 5 篇公式密集论文。
- 5 篇多栏/图注复杂论文。
- 若做 OCR，本地再准备 5 篇扫描版或低文本层 PDF。
- 5 篇含大量脚注、作者/机构、人名实体或页下注释的论文。

对比组：

- 当前纯文本固定页分块版。
- 增加术语记忆版。
- 增加结构感知分块版。
- 增加表格保护版。
- 增加图像/OCR 路由版。
- 增加结构化 HTML/PDF 输出版。
- 增加 QA 局部修复版。

指标：

- 漏译率。
- 术语一致率。
- 表格行列保持率。
- 图注/脚注归位准确率。
- 图像/扫描页覆盖率。
- 页面交接碎片率。
- 数字/公式/引用保留率。
- 英文残留比例。
- 人工可读性评分。
- 单篇耗时、API 调用次数、估算成本。
