## 重要说明

本文是工程与方案梳理，不等同于专利法律意见。真正申请前，需要专利代理师围绕新颖性、创造性、实用性做检索、交底书整理和权利要求撰写。

## 基本判断

如果只把项目描述为“PDF 翻译工具”“AI 文献翻译方法”“使用术语表的翻译系统”或“保留格式的文档翻译方法”，很容易落入已有技术。更适合收束为一个具体技术方案：

> 针对学术 PDF 长文翻译中的上下文压缩、术语漂移、结构错乱和表格线性化问题，构建一种结构感知的层级式翻译控制方法：先将文档解析为带类型与边界的中间结构，再按结构原子和上下文预算生成翻译块，翻译过程中动态更新术语、摘要和衔接记忆，并通过结构不变量进行质量检测与局部重译修复。

当前更推荐把“算法级翻译流程”作为专利主线，而不是把“完全不借助 AI 的本地翻译工具”作为主线。OCR、图像识别、本地模型和 PDF 输出应作为支撑模块，服务于结构识别、低依赖运行和结果可读性。

## 当前核心问题

| 问题 | 当前表现 | 专利化切入点 |
| --- | --- | --- |
| 表格线性化 | 表格文字被抽成普通文本，行列、表头、单位和脚注丢失 | 表格结构保护、单元格上下文构造、表格维度 QA、单元格锁定 token QA |
| 注释/人名/图注归属丢失 | 脚注、图注、人名机构和正文混在一起 | 基于结构类型和父子关系的内容归位 |
| 图像与扫描页处理不足 | 仅靠文本数量估计是否需要图像模型，容易误判 | 本地 OCR + 页面截图 + 图像区域 + VLM 路由 |
| 分页切块造成碎片 | 页末半句、跨页表格、跨页图注被拆开 | 结构单元分段和跨块顺延记忆 |
| Markdown 可读性不足 | 复杂表格、脚注、图文混排在 Markdown 中表现差 | 结构化 Markdown、双语 HTML、PDF 译文生成 |

## 现有技术拥挤区

以下方向不宜单独作为核心创新点：

- 机器翻译结合术语表或翻译记忆。
- 普通上下文感知翻译记忆。
- 文档翻译后保留样式或原位置回填。
- 单纯把表格单元格拿出来翻译。
- 单纯保留 PDF 原排版、公式和图表。

这些能力可以作为系统组成部分，但不应作为唯一创新点。

## 建议核心名称

- 一种面向学术 PDF 的结构感知层级式机器翻译方法及系统。
- 一种基于结构中间表示和动态记忆反馈的学术 PDF 分层翻译方法。
- 一种基于结构不变量检测与局部修复的学术文档翻译方法。
- 一种基于本地结构识别与结构单元分段的学术 PDF 翻译排版方法。

## 建议技术流程

```text
输入 PDF
  -> 本地结构解析与 OCR 风险检测
  -> 生成轻量 DocumentIR
  -> 译前巡视：术语、章节、表格/公式/图注/扫描页风险
  -> 用户确认术语与策略
  -> 结构单元分段：段落、表格、公式、图注、脚注不可切分
  -> 分层翻译：正文、表格、图注、公式附近文本、注释分别处理
  -> 动态记忆更新：术语、实体、摘要、尾部衔接
  -> 结构不变量 QA：数字、公式、引用、表格维度、术语一致性
  -> 局部修复：重译高风险块、段落或单元格
  -> 输出结构化 Markdown、双语 HTML、PDF 译文、QA 报告和资料包
```

## 可形成权利要求的组合方向

### 方向 A：结构感知层级翻译控制

**核心组合**：PDF 结构解析、结构原子标注、自适应分块、块间摘要、顺延记忆、翻译后记忆回流。

**技术效果**：
- 降低长上下文压缩导致的漏译。
- 降低跨块指代断裂。
- 降低术语漂移。
- 保持任务可恢复、可局部重译。

### 方向 B：表格保护式学术翻译

**核心组合**：表格结构识别、单元格上下文构造、数字/单位/符号锁定、表格维度 QA、异常单元格局部重译。

**技术效果**：
- 避免表格被线性化后语义错乱。
- 保持行列关系、表头和单位上下文。
- 提高表格短文本翻译准确性。

### 方向 B2：注释、图注与实体归位

**核心组合**：脚注/页下注释检测、图注/表注归属、人名/机构/数据集/模型名实体识别、正文与注释分层输出。

**技术效果**：
- 减少注释混入正文造成的阅读断裂。
- 保留人名、机构和专名上下文。
- 提高 Markdown、HTML 和 PDF 输出的可读性。

### 方向 C：动态记忆反馈式文档翻译

**核心组合**：译前术语候选、用户确认或置信度标记、翻译中术语回流、按页码/章节/结构类型选择性注入、冲突检测。

**技术效果**：
- 越翻译越稳定。
- 减少专业术语前后不一致。
- 降低上下文窗口浪费。

### 方向 D：结构不变量驱动的局部修复

**核心组合**：从源 DocumentIR 提取不变量、译后比对、定位异常块、选择修复粒度、只重译局部。

**技术效果**：
- 降低修复成本。
- 提高译文完整性。
- 避免全文重译引入新不稳定。

### 方向 E：图像页与 OCR/VLM 路由

**核心组合**：文本层统计、页面截图、图片区域检测、OCR 置信度、图注识别、VLM 调用决策、OCR 结果回写门禁、OCR 候选 QA、OCR 候选晋级和结构分段输入副本。

**技术效果**：
- 避免扫描页和图片型表格漏译。
- 减少对图像模型的无效调用。
- 让图像处理结果回流到结构 IR，而不是只生成零散描述。

## 最建议优先落地的专利原型

优先做 “A + B + B2 + D”，并把 “E” 作为支撑能力的最小闭环：

```text
DocumentIR
  -> 表格保护
  -> 注释/图注/实体归位
  -> 图像/OCR 风险路由
  -> 结构单元分段
  -> 动态记忆注入
  -> 结构不变量 QA
  -> 局部修复
  -> 双语 HTML / PDF 译文输出
```

原因：

- 与现有项目距离最近，不需要推翻当前管线。
- 能直接回应固定页分块和表格错乱问题。
- 比单纯术语记忆更有技术形态。
- 更容易用实验数据证明效果。
- 能把本地 OCR 和 PDF 输出纳入算法流程，而不是另起一条孤立路线。

## 原型模块要求

### DocumentIR

建议最小结构：

```json
{
  "doc_id": "paper-id",
  "pages": [
    {
      "page_no": 1,
      "blocks": [
        {
          "block_id": "p1-b3",
          "type": "paragraph | heading | table | table_cell | caption | footnote | formula | reference | person | organization | header_footer | image",
          "bbox": [0, 0, 0, 0],
          "text": "...",
          "parent_id": null,
          "order": 3,
          "locked_tokens": ["(1)", "F1-score", "Table 2"],
          "meta": {
            "parent_relation": "caption_for_table | caption_for_figure | footnote_for_table | footnote_for_block",
            "parent_warning": "orphan_caption | orphan_footnote",
            "entities": [
              {
                "type": "person | organization | model_or_dataset | acronym",
                "text": "BERT",
                "confidence": "low | medium | high",
                "source": "organization_suffix | model_dataset_pattern | citation_author_year | uppercase_acronym"
              }
            ]
          }
        }
      ]
    }
  ]
}
```

验收：同一份 IR 能支撑分块、翻译、QA、双语输出和局部修复。

### 表格保护

必须保留：

- 行列结构。
- 表头、行名、单位、脚注。
- 跨页续表关系。
- 数字、显著性符号、公式片段。
- 单元格上下文提示。
- 译后行列数和数字集合比对。

验收：表格密集论文的表格行列保持率、续表风险和表格脚注归属可统计，异常能定位到表格、续表边界或单元格。

### 结构感知分块

分块依据：

- 结构原子不可切分。
- 模型上下文预算。
- 章节和段落边界。
- 表格/公式/图注风险。
- 脚注、注释和人名机构上下文。
- 跨页续段和续表。
- 跨块顺延记忆。

验收：相比固定页滑窗，重复译文、半句切断、术语漂移和漏译风险下降。

### QA 与局部修复

优先做规则 QA：

- 数字集合不变量。
- 公式/变量名不变量。
- 引用编号不变量。
- 表格维度不变量。
- 图表编号不变量。
- 实体候选保留不变量。
- 英文残留比例。
- 术语一致性。
- 重复译文检测。

验收：QA 能输出 `qa_report.md`，并能给出重译块、段落或单元格建议。

本轮补充：局部修复闭环已从“候选片段验证”推进到“独立合并副本与合并后 QA”。管线会生成 `repair_merge.json` / `repair_merge.md`、`output/repaired_chunks/`、`output/repaired_full.md`、`repair_merge_qa.json` / `repair_merge_qa.md`，并把修复合并应用率、合并后 issue 降幅等指标写入 `experiment_metrics.json`。该能力仍不覆盖正式译文 chunk，而是作为补丁合并和失败回退的专利原型证据。

本轮补充：PERF-01 基础已落地，并已补入翻译块级 HTTP 尝试/重试证据。管线会生成事件级 `run_log.jsonl` 和汇总级 `run_metrics.json`，记录 DocumentIR、结构 QA、表格重建、OCR/VLM 路由、分段、每块翻译请求、增量合并、最终合并、译后 QA、局部修复、双语 HTML 等阶段耗时，并汇总请求字符数、译文字符数、估算 token、翻译请求次数、HTTP 尝试次数、HTTP 重试次数和吞吐指标；`cost_estimate.json` 会读取可配置成本画像，按估算 token、字符数和可计费请求次数计算金额，其中 `per_request` 成本优先使用 HTTP 尝试次数；`experiment_metrics.json.performance` 会回链这些运行与成本指标，用于后续做同一论文在固定页分段、结构分段、表格保护、QA 修复等不同流程之间的耗时/成本对比。当前不硬编码会变化的 DeepSeek/OpenAI/DeepL 价格；失败尝试的真实 token/字符计费、巡视/修复请求成本和服务商账单校准仍需后续补充。

本轮补充：图像/OCR 路由已从“文本与图片统计建议”推进到“页面预览 + 候选区域裁剪证据 + OCR 调度任务 + 可选本地 OCR 执行 + OCR 结果载荷 + OCR 结果回写门禁”。非 `text_only` 页面会在 `output/vision_pages/` 下生成页面预览图，并按 `image`、`table`、`caption`、`formula` 等 DocumentIR 块在 `output/vision_crops/` 下生成候选区域裁剪图；两类证据都会由 `vision_route.json.pages[].evidence` 回链。`output/ocr_tasks.json` 会继续把这些可视证据转成待执行 OCR 任务，记录输入图、优先级、建议本地引擎、VLM fallback、任务状态和 DocumentIR 回写目标。CLI 可通过 `--execute-ocr` 调用本地 tesseract CLI 执行这些任务，或通过 `--ocr-results` 回灌外部 OCR / 后续本地 OCR 执行器返回的 `ocr-results-v1` 载荷；`output/ocr_results.json` 会记录来源、结果数、无效结果数、状态分布、引擎分布以及本地执行尝试数、成功数、失败数和跳过数。`output/ocr_writeback.json` 与 `output/document_ir_ocr.json` 会把 OCR 结果载荷通过状态、文本和置信度门禁后追加为 page/block 的 `meta.ocr_candidates`，默认无真实 OCR 结果时记录 pending 任务，不污染正式源文本和翻译块。`experiment_metrics.json` 会统计 `vision_preview_page_count`、`vision_preview_page_rate`、`vision_region_crop_count`、`vision_region_crop_per_routed_page`、`ocr_task_count`、`ocr_task_per_routed_page`、`ocr_ready_task_rate`、`ocr_result_payload_count`、`ocr_invalid_result_count`、`ocr_executor_attempted_task_count`、`ocr_executor_success_rate`、`ocr_task_result_coverage_rate`、`ocr_result_acceptance_rate` 和 `ocr_writeback_apply_rate`，资料包会把预览图、区域裁剪图、OCR 调度任务、OCR 识别结果、OCR 结果回写和 OCR 增强 IR 分别归档到 `质量/图像OCR页面预览/`、`质量/图像OCR区域裁剪/`、`质量/OCR调度任务.json`、`质量/OCR识别结果.json`、`质量/OCR结果回写.json` 与 `设置/OCR增强文档结构IR.json`。这让后续对扫描页、图片型表格和图注复杂页的处理有可视证据、可执行契约、可替换执行器、可审计输入载荷和可审计回写链路，而不只是抽象风险分数。

本轮补充：批量实验证据链基础已落地。`pdf_translate.experiments` 和 CLI `pdf-translate experiment` 可以把多篇 PDF 与 `page`、`structure`、`structure+ocr`、`structure+repair` 等策略组成实验矩阵，逐次调用现有单篇管线，并汇总每次的 `experiment_metrics.json`、`run_metrics.json` 与 `cost_estimate.json`。批量输出包含 `batch_experiment_manifest.json`、`batch_experiment_summary.json` 和 `batch_experiment_summary.md`，其中记录单次运行状态、指标文件路径、策略均值和相对基线差值。该能力解决了“只有单任务指标、缺跨样本汇总器”的证据缺口；下一步仍需要按申请前测试集实际跑样本、录入人工可读性评分并校准真实账单成本。

## 申请前证据清单

建议准备测试集：

- 10 篇普通英文论文。
- 10 篇表格密集论文。
- 5 篇公式密集论文。
- 5 篇多栏/图注复杂论文。
- 若做 OCR，本地再准备 5 篇扫描版或低文本层 PDF。
- 5 篇含大量脚注、作者/机构、人名实体或页下注释的论文。

对比组：

- 当前纯文本固定页分块版。
- 增加术语记忆版。
- 增加结构感知分块版。
- 增加表格保护版。
- 增加图像/OCR 路由版。
- 增加结构化 HTML/PDF 输出版。
- 增加 QA 局部修复版。

指标：

- 每次任务的 `experiment_metrics.json`，用于回链结构 QA、表格重建、OCR/VLM 路由、OCR 候选 QA、OCR 候选晋级、译后 QA、局部修复计划、局部修复请求、局部修复结果、局部修复验证、局部修复合并、合并后 QA 和运行性能指标。
- 每次任务的 `run_metrics.json` / `run_log.jsonl`，用于记录阶段耗时、翻译请求次数、HTTP 尝试/重试次数、字符数、估算 token、吞吐和跳过块。
- 每次任务的 `vision_route.json`、`vision_pages/`、`vision_crops/`、`ocr_tasks.json`、`ocr_results.json`、`ocr_writeback.json`、`ocr_candidate_qa.json`、`ocr_candidate_promotion.json`、`document_ir_ocr.json` 与 `document_ir_promoted.json`，用于说明哪些页面进入 OCR/VLM 候选、原因是什么、对应页面预览和候选区域裁剪证据在哪里、后续本地 OCR/VLM 任务应如何执行、本地 OCR 是否尝试/成功/失败、实际 OCR 结果载荷是什么、OCR 候选文本是否通过门禁、以及哪些候选最终进入结构分段输入副本。
- 每次任务的 `cost_estimate.json`，用于记录成本画像命中情况、币种、单位价格、估算 token/字符/请求用量、可计费请求数和估算金额。
- 漏译率。
- 术语一致率。
- 表格行列保持率。
- 表格可重建率与单元格级数字/单位/显著性 token 覆盖。
- 表格形状异常率。
- 表格单元格 token 异常率与锁定 token 缺失率。
- 图注/脚注归位准确率。
- 图像/扫描页覆盖率。
- 页面交接碎片率。
- 页面边界切开率 / 保护率。
- 结构分段相对固定页分段的边界切开降幅。
- 数字/公式/引用保留率。
- 英文残留比例。
- 人工可读性评分。
- 单篇耗时、API 调用次数、HTTP 重试次数、估算 token、吞吐和估算成本；当前已具备耗时/字符/token 估算、HTTP 尝试回写与可配置价格画像，失败尝试 token 成本仍需结合服务商账单校准。
